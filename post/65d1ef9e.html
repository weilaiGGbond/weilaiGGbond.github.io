使用 vuepress 创建静态站点预构建

静态站点定义

静态站点（SSG，全称为 Static Site Generation）也称为预渲染，所有页面都是构建时生成和数据渲染的，即一次生成可被多次访问。
优点在于页面响应快，托管和运维成本低；但缺点也十分明显，页面数据不具备实时性，难以应对交互复杂的页面，适用于数据时效性要求不高的场景。

SSG、SSR、CSR 以及同构渲染的区别

  • SSG（静态站点生成）：如上述介绍，构建时生成页面并渲染数据。
  • CSR（客户端渲染）：常见的 SPA（单页应用）渲染方式就是采用 CSR 模式进行渲染的。其特点是所有页面都在客户端，不同页面间切换流畅，有利于前后端分离，提高开发效率；缺点是首次访问的响应时间长，且不利于 SEO 的查找。
  • SSR（服务器端渲染）：与 CSR 相对，当页面访问某个页面时，由服务器端生成 HTML 进行数据渲染，再由浏览器解析呈现给用户。优点是首次访问响应时间短，利于 SEO 搜索；但是会给服务器造成较大的压力，且不利于前后端分离。
  • 同构渲染（CSR + SSR 渲染模式）：在网站的首页采用 SSR，其他页面采用 CSR 渲染模式，可以解决 CSR 不利于 SEO 搜索的问题，同时也解决了 SSR 带来的服务器压力过大的问题；但缺点是两者渲染方式配置繁琐，难度较大。

创建 VuePress 项目

使用 pnpm create vuepress vuepress-starter 命令创建项目，这里采用的下载依赖的命令行工具是 pnpm，其优点如下：

  1. 存储方式：
    • npm：每次安装包时，都会在 node_modules 目录下创建一个完整的依赖树，可能会导致重复的包占用更多空间。
    • pnpm：使用全局存储（类似硬链接），只存储每个包的一个副本，减少了磁盘空间的使用并加快了安装速度。
  2. 安装速度：
    • npm：依赖解析和安装速度较慢，尤其是在大项目中。
    • pnpm：由于其优化的存储方式，通常安装速度更快，特别是在多个项目中共享依赖时。
  3. 依赖的解析方式：
    • npm：使用扁平化的依赖结构，可能导致不同包间的依赖冲突。
    • pnpm：严格按照依赖树安装，确保每个包都能找到所需的确切版本，从而减少冲突。
