ORM

ORM 是什么？

ORM 对象关系映射，其主要作用是在编程中，把面向对象的概念跟数据库中的概念对应起来

Object —- Mapping —- DB

ORM 特点

方便维护，数据模型定义在同一个地方，利于重构

代码量少、对接多种库，代码逻辑更易懂

工具多、自动化能力强，数据库删除关联数据、事务操作等

缺点：由于高级集成、自动化能力强，ORM 的封装性更强，所以对于代码优化以及性能优化来说比较困难

数据库相关概念

关系型数据库，是指采用了关系模型来组织数据的数据库 MySql

NoSQL 是对不同于传统的关系数据库的数据库管理系统的统称 Mongodb

常见关联模型

  • 一对一
  • 一对多
  • 多对多

关系型数据库特点

优点：易于维护、使用方便、支持复杂查询效率高

缺点：读写性能差，灵活性差

适用场景：各类业务系统、管理系统、安全性较高的场景

非关系型数据库特点

优点：易于扩展，大文件存储、查询速度快

缺点：复杂计算与联合查询效率低

适用场景：多格式&海量数据、分布式消息系统、统计排行
